# 📝 内置提示词指南

## 概述

LLMSider 提供了一系列预配置的提示词，帮助您快速高效地完成常见任务。这些提示词针对各种用例进行了优化，并支持多语言。

---

## 🚀 快速开始

### 访问内置提示词

1. **打开聊天界面**
2. **点击提示词图标**（📝）或使用快速选择器
3. **搜索或浏览**可用的提示词
4. **选择提示词**即可立即使用

### 使用提示词

```
方法 1: 快速选择器
- 在聊天输入框中按 `/`
- 输入搜索提示词
- 选择并应用

方法 2: 提示词按钮
- 点击输入框上方的 📝 图标
- 浏览分类
- 点击应用
```

### 管理提示词

您可以在 **设置 → LLMSider → 提示词管理** 中自定义您的提示词库:
- **添加自定义提示词**: 创建您自己的可重用提示词
- **隐藏内置提示词**: 切换您不使用的系统提示词的可见性
- **编辑/删除**: 管理您的自定义提示词

## 📚 可用提示词

### 写作与编辑

#### 继续写作
**ID**: `builtin-continue-writing`  
**说明**: 从你停下的地方继续写作  
**用例**: 扩展文章、故事或文档

**示例:**
```
[您的文字]
---
[AI 从您的文字自然延续]
```

#### 修正语法和拼写
**ID**: `builtin-fix-grammar-and-spelling`  
**说明**: 纠正语法、拼写和标点符号  
**用例**: 润色您的写作

**示例:**
```
输入: "我写了这个句子错误的"
输出: "我把这个句子写错了"
```

#### 改进写作
**ID**: `builtin-improve-writing`  
**说明**: 增强清晰度、风格和可读性  
**用例**: 专业沟通、博客文章

---

### 翻译

#### 翻译成中文
**ID**: `builtin-translate-to-chinese`  
**说明**: 将任何文本翻译成中文  
**语言**: 支持所有主要语言 → 中文

#### 翻译成英文
**ID**: `builtin-translate-to-english`  
**说明**: 将任何文本翻译成英文  
**语言**: 支持所有主要语言 → 英文

---

### 内容生成

#### 总结
**ID**: `builtin-summarize`  
**说明**: 创建长文本的简明摘要  
**用例**: 研究论文、文章、会议笔记

**输出格式:**
- 关键点（3-5 个要点）
- 主要结论
- 1-2 句话摘要

#### 像给 5 岁孩子解释
**ID**: `builtin-explain-like-im-5`  
**说明**: 简化复杂主题  
**用例**: 学习、教学、文档

#### 生成目录
**ID**: `builtin-generate-table-of-contents`  
**说明**: 从文本创建结构化目录  
**输出**: Markdown 格式的目录

#### **新功能** 生成 Obsidian Meta 信息
**ID**: `builtin-generate-obsidian-meta`  
**排序**: 1300  
**说明**: 为 Obsidian 笔记生成 YAML front matter

**功能特点:**
- 分析内容并提取关键信息
- 生成适当的标签、别名和元数据
- 遵循 YAML 最佳实践（snake_case 键名、正确引号）
- 智能分类和摘要
- 不会无中生有缺失的信息

**输出格式:**
```yaml
---
tags: [标签1, 标签2, 标签3]
aliases: [别名1, 别名2]
author: 作者名
source: 来源 URL 或引用
category: 分类
summary: 简短摘要（1-2 句话）
---
```

**YAML 格式规则:**
- 键名使用 `snake_case`（小写字母加下划线）
- 使用空格缩进（不使用 tab）
- `key: value` 格式
- 列表项使用连字符
- 包含特殊字符的字符串需要引号
- 不包含 `title`、`created`、`modified` 或 `status` 字段

**使用场景:**
- 组织研究笔记
- 标记导入的文章
- 分类会议笔记
- 结构化知识库条目

## 🎨 自定义

### 创建自定义提示词

1. **转到设置** → LLMSider → 提示词管理
2. **点击"添加新提示词"**
3. **填写**:
   - 标题: 描述性名称
   - 说明: 功能描述
   - 内容: 提示词模板
   - 标签: 用于组织
4. **保存**

### 提示词模板

在自定义提示词中使用占位符:

```markdown
分析以下 {内容类型}:
{选中文本}

重点关注: {关注领域}
```

**可用占位符:**
- `{选中文本}` - 当前选中的文本
- `{当前文件}` - 活动文件内容
- `{剪贴板}` - 剪贴板内容
- 您定义的自定义变量

## 💡 最佳实践

### 何时使用内置提示词

✅ **适合:**
- 重复性任务
- 标准操作
- 快速转换
- 一致的格式化

❌ **不太适合:**
- 高度特定的一次性任务
- 复杂的多步骤工作流
- 需要大量上下文的任务

### 组合提示词

**示例工作流:**
1. 对研究文章使用"总结"
2. 对摘要使用"提取行动项"
3. 使用"生成 Obsidian Meta"进行组织
4. 使用"创建大纲"制作演示文稿

### 提示词链式调用

```
步骤 1: 翻译成英文 → 结果 A
步骤 2: 总结结果 A → 结果 B  
步骤 3: 为结果 B 生成 Meta → 最终结果
```

## 🔧 高级功能

### 提示词排序

提示词根据以下因素排序:
1. **排序索引**（较低 = 较高优先级）
2. **最后使用**（最近使用的优先显示）
3. **字母顺序**（后备排序）

### 搜索和筛选

**搜索提示词:**
- 按标题: 输入提示词名称
- 按说明: 输入用例
- 按标签: 使用 `#标签` 语法

**快速访问:**
- 固定常用提示词
- 分配键盘快捷键（设置 → 快捷键）

## 🆘 故障排除

### 提示词未显示

**检查:**
- 提示词已启用（设置 → 提示词管理）
- 没有激活的筛选器
- 搜索查询正确
- 插件是最新版本

### 提示词不工作

**验证:**
- 已选择并激活模型
- API 密钥有效
- 提示词语法正确
- 没有占位符错误

### 自定义提示词问题

**常见问题:**
- 缺少必需的占位符
- 提示词元数据中的 JSON 无效
- ID 重复
- 循环引用
